ARRAYS are ordered, integer-indexed collections of any object. Array indexing starts at 0. The objects in an array are counted from a left to right index starting at 0. The first object is at index 0, second object is at index 1. You can also count array objects from right to left using a negative number index starting at -1 (no zero.) A new array can be created by using the literal constructor []. Arrays can contain different types of objects. For example, an array can contain an Integer, a String and a Float or any variation of these objects/elements. The word object is indicative of the thrust of the Ruby programming language and object orientation. Everything in Ruby is an object. The objects in Ruby Arrays are generally associated with data containment and relational data. Here are some examples of the standard structure of simple Ruby arrays.

HASHES are great for data containment. Sometimes arrays are the better choice, other times hashes. Which to choose! Hashes are different than arrays. Hash is a dictionary-like collection of unique keys and their values. Also called associative arrays, they are similar to Arrays, but where an Array uses integers as its index, a Hash allows you to use any object type. Hashes enumerate their values in the order that the corresponding keys were inserted. Hashes can be created using their implicit form. Each hash contains pairs of dependent information know as key/value. Each hash contains only of each key element type which is why it is called unique. See first hash for pets. I can only have 1 fuzzy. But I can have another key with the same value. So the fuzzy key has a value of 6 (age) and the slate key can also have a value of 6 (age). Hashes, like arrays, offer syntax and structure variations depending on what I need to do, or code I will build. A Hash can also be created through its ::new method:
