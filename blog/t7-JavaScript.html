Click on the underlined "FizzBuzz" above for a link to a blog that was my introduction to the relevance of the code block FizzBuzz as a interview challenge for programmers and its association with a UK children's game. After I read the blog I went to Wikipedia for additional information. I found an example of the code but it involved more code structure than I was familiar with or capable of executing. So I set about building the code based on my learning to date. Since I only had 3 weeks of Ruby learning the first example represents what I was able to produce. It is a concise, and basic if, elsif, else construct that enables a block of code to repeat itself over and over until a condition is met. It runs as long as the need for the code remains 'true'. Once the code goes over all numbers and accomplished its task, and the condition for its need is now false, the code stops. The first line begins with creating a varirable numbers and giving it a value of a range of numbers from 1 to 100. Because of what will takes place next the range of numbers must be converted to an array for execution. The second line is a line of code that sets in place the iteration process by instructing the code to go through 'each' number from 1 to 100 and stop at 100. This next few lines begin the looping and selection of which of the next four possibilities to execute. The displayed result will depend on which numbers are evenly divisible by 3, 5, and 15, and the output will be one of three words. Or else just the number itself if not divisible evenly.

The second example of FizzBuzz is standard code configuration of another programming language called JavaScript. As with the Ruby example, JavaScript also utilizes an iteration process that loops, or repeats the same block of code on the numbers from 1 to 100. As you look at both blocks of code the similarities immediately stand out. The general structure of the inner blocks with the if loop in both languages are almost the same except for the slight deviation in spelling. The Ruby loop uses an if, elsif, else contruct while JavaScript uses a barely different if, else if, else construct. A slight spelling difference. Otherwise they do the same thing with identical results. But it is the first two lines of code where the contrast is apparent. Ruby implies number is a variable based on standard practice while JavaScript clarifies using 'var' the number is a variable. Then Javascript has a language standard three part code block that is enclosed in parantheses. In the short time I have spent with JavaScript I have become aware of the frequency of punctutation to enclose lines or blocks of code with semi-colons, braces, commas, etc... Whereas, Ruby is more language based with less use of punctuation barriers. Ruby still employs some punctutaion for code differentiation but not to the extent of JavaScript.

In conclusion, I will have more to say about JavaScript in the future. But after just 5 days I have barely begun my understanding of its complexity and nuances. And I am just a little further in a long journey to properly understand and practice the features and details of the programming language Ruby. There is so much more to learn. And whoever came up with the word FizzBuzz, I think its great.
